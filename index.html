security and privacy

with a standard rails application I do need some extra work to get it more secure. this template tries to do that extra work for you. after a nice security is in place some we can install some gems which helps to keep person related data well protected:

  • keep the person related data on the server
  • expire person related data after some time
  • tell the browser how to cache person related date (:public, :protected, :private)
  • tell the proxies between the browser and the server how to cache data
  • expire the browser session after some idle time
  • guard the action an user can perform on the server (controller#action in rails speak) to allow only authorized users to access certain person data
  • enforce SSL for all communication, to protect the data over the wire and prevent session highjacking with tools like firesheep

rails templates

the rails templates below will setup a rails application which uses ixtlan-users as authentication server. all the above issues will be (more or less) addressed.

ixtlan client with erb templates


$ rails new my_app -m http://mkristian.github.com/ixtlan-users/ixtlan.rb

ixtlan client with GWT frontend and erb templates


$ gem install ruby-maven
$ gwt new my_app com.example --menu --session -- -Dtemplate=http://mkristian.github.com/ixtlan-users/ixtlan-gwt.rb
